# Vision

Build a chat-first workflow where the user can talk through ideas naturally, have the agent write tentative notes on disk as the conversation happens, and later resume from durable local context instead of retyping everything into a website.

The workflow should:
- favor small, well-scoped tasks after the idea is clear
- keep context visible on disk
- avoid runaway agent activity
- use Kanban only when execution should start
- make the repo itself the source of truth for what the user meant, what changed, and what still needs work
- keep one small next action ready instead of forcing the user to re-explain the whole project after every restart
