# Brain

This directory is the durable on-disk memory for chat-first work in this repo.

Rules:
- Always write while we talk.
- Treat notes as tentative unless explicitly marked decided.
- Read `ARCHITECTURE_CONVERSATION_LOG.md` first before planning or refactors.
- Read `brain/` first on resume.
- Kanban is for execution only, not for figuring out intent.
- Keep exactly one small next action in `brain/NEXT_STEPS.md`.

Files:
- `VISION.md`: the evolving product direction.
- `CURRENT_STATE.md`: what is true right now.
- `DECISIONS.md`: settled choices that should not be re-litigated.
- `OPEN_QUESTIONS.md`: unresolved items that still need answers.
- `NEXT_STEPS.md`: one concrete next move, kept small.
- `CONVERSATIONS/`: dated notes from live discussion.
- `TASK_DRAFTS/`: short task specs distilled from conversation.
- `PROJECT_MAP.md`: high-level overview of the whole codebase.
- `ARCHITECTURE_CONVERSATION_LOG.md`: required first-pass architecture context before planning or refactors.
